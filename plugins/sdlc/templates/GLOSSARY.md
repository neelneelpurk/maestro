# ai-sdlc glossary

The vocabulary, labels, and state machine of the pipeline. Generated by
`/sdlc:init`; the conventions in `.claude/rules/sdlc.md` reference it.

## Branches & PRs

- **Default branch** (`main`/`master`) — the trunk. Only ever updated by a human merging an integration PR (or a `ship` PR).
- **Integration branch** (`sdlc/integration-<stamp>`) — a per-run branch off the default branch where `drain`/`auto` land each issue. Created by `integration.sh start`.
- **Issue branch** (`sdlc/issue-<n>-<slug>`) — one per issue, created with `gh issue develop` so GitHub records a native branch↔issue link. Branches off the integration branch (drain/auto) or the default branch (ship).
- **Integration PR** — integration branch → default branch. The **single human review gate**; never auto-merged. Accumulates a progress comment + checklist of integrated issues.
- **Per-issue PR** — issue branch → integration branch; auto-merged once green.
- **Ship PR** — issue branch → default branch, with `Closes #<n>`; awaits human review.

## Labels

### Type
| Label | Meaning |
|---|---|
| `prd` | A PRD / epic parent issue. |
| `roadmap` | A roadmap parent issue. |
| `bug` / `enhancement` | Category. |
| `tech-debt` | Technical debt to pay down. |

### Gate / mode
| Label | Meaning |
|---|---|
| `ready-for-agent` | Human-approved for autonomous implementation (the human gate). |
| `auto` | Autonomous lane — created by `/sdlc:auto`; **skips the `ready-for-agent` gate**; picked automatically. |
| `hitl` | Needs a human; never auto-picked. |

### Execution state (one at a time)
| Label | Meaning |
|---|---|
| `in-progress` | A worker is implementing it. |
| `in-review` | `ship`: a PR to the default branch is open, awaiting your review. |
| `waiting-for-human-closure` | `drain`/`auto`: the work is merged into the integration branch; the issue stays open until you merge the integration PR and close it. |
| `blocked` | Board marker: has open dependencies. |

### PR
| Label | Meaning |
|---|---|
| `integration` | The integration → default-branch PR (your review gate). |

## State machine

```
created ──(human)── ready-for-agent ─┐        (supervised lane)
        └─(/sdlc:auto)── auto ────────┤        (autonomous lane; skips the gate)
            assigned to me, unblocked  ▼
                       picked → in-progress
        /sdlc:ship → PR → default branch → in-review            → you merge → closed
        drain/auto → PR → integration   → auto-merge (green)    → waiting-for-human-closure
                                          └ you merge the integration PR → bulk-close
        hitl ───────────────────────────────► never auto-picked (a human implements)
```

## Dependencies

Modeled with **native GitHub issue dependencies** (`blocked_by`). A blocker is
considered cleared once it is **closed** or labelled `waiting-for-human-closure`
(its work is already in the integration branch) — which is what lets the drain
queue advance without auto-closing issues.
